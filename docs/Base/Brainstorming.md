# Brainstorming

## Introdução

<div align="justify">
O brainstorming foi uma técnica escolhida para a elicitação de alguns requisitos. Esta técnica permite gerar ideias de forma criativa e colaborativa para resolver problemas e identificar requisitos para um projeto.
</div>

## Metodologia

<div align="justify">

1. O primeiro passo foi analisar entender definir um tema do brainstorming, nesse caso o tema escolhido foi "Dificuldades de vender produtos usados online". Durante essa fase,foram definidos os principais problemas e dificuldades relacionados.

2. Depois de especificar os problemas a serem resolvidos, o brainstorming feito usando o Miro como ferramenta online. Nessa plataforma foram adicionados post-its para representar os problemas a serem resolvidos e as ideias de solução.

3. Com base nas ideias coletadas no Miro, foi possível elicitar os alguns requisitos para o nosso projeto.
</div>

### Requisitos Elicitados

### Requisitos Não Funcionais

| <p align="center">ID</p> |                         <p align="center">Requisito</p>                          |
| :----------------------: | :------------------------------------------------------------------------------: |
|          RNF01           | O usuário deve ser capaz de cadastrar um produto de maneira simples e intuitiva. |
|          RNF02           |        Preço da plataforma deve ser atraente para vendedores iniciantes.         |

### Requisitos Funcionais

| <p align="center">ID</p> |                                             <p align="center">Requisito</p>                                             |
| :----------------------: | :---------------------------------------------------------------------------------------------------------------------: |
|           RF01           | Sistema deve possuir uma integração que pre-preencha informações de um produto baseados no seu código de barras ou SKU. |
|           RF02           |                  Vendedor deve ser capaz de selecionar quais métodos de pagamento ele deseja oferecer.                  |
|           RF03           |                Vendedor deve ser capaz de selecionar parceiros da plataforma para realização da entrega.                |
|           RF04           |               O usuário deve ser capaz de escolher o método de pagamento entre os aceitos pelo vendedor.                |
|           RF05           |              O usuário deve ser capaz de visualizar preço e estimativa de tempo de entrega de um produto.               |
|           RF06           |                     Vendedor deve ser capaz de gerar e avaliar a descrição de um produto usando IA                      |
